[시스템]
현재 날짜: {current_date}
현재 요일: {current_weekday}

날짜 계산 규칙:
- '오늘' → {current_date}
- '내일' → {current_date} + 1일
- '모레' → {current_date} + 2일
- '이번 주 X요일' → 이번 주의 해당 요일 날짜
- '다음 주 X요일' → 다음 주의 해당 요일 날짜
- '다음주' → 다음 주 월요일
- 'X월 Y일' → 해당 연도의 X월 Y일

[지시]
[입력 전문]만을 근거로 "tasks" 필드를
요청된 **JSON 형식**으로만 출력하라.
**다른 설명이나 텍스트, 마크다운(` ```json `)을 포함하지 말고, 순수한 JSON 객체 하나만 생성해야 한다.**

[입력 전문]
{transcript}

{{
  "tasks": [
    {{
      "description": "태스크 내용",
      "assignee": "담당자 이름",
      "due": "마감일 표현",
      "due_date": "YYYY-MM-DD"
    }}
  ]
}}

[규칙]
1. 반드시 유효한 JSON 객체 하나만 출력한다.
2. tasks는 회의에서 언급된 모든 할일, 후속 처리해야할 업무이다.
3. tasks는 개수 상관없이 모든 태스크를 뽑는다.
4. tasks를 뽑을 경우, 도메인 분야에만 국한되지 말고, 기본적인 태스크도 모두 추출한다.

5. tasks 필드 상세 규칙:
   - description: 태스크 내용 (전문에서 그대로 추출)
   - assignee: 태스크 수행 대상 (전문의 화자 이름 추출, 예: test1, test2, test3)
   - due: **반드시** 전문에 나온 마감일 표현을 그대로 추출
     * 예시: "내일중", "다음주 수요일까지", "이번주 금요일", "12월 20일까지", "오늘 오후"
     * 마감일 언급이 없으면 null
     * **절대 변형하지 말고 전문의 표현 그대로 사용**
   - due_date: due를 [시스템]의 날짜 계산 규칙으로 변환한 YYYY-MM-DD 형식
     * "내일중" → {current_date} + 1일 계산
     * "다음주 수요일까지" → 다음 주 수요일 날짜 계산
     * "이번주 금요일" → 이번 주 금요일 날짜 계산
     * due가 null이거나 계산 불가 → null

6. due와 due_date 추출 예시:
   - 전문: "test2님, 내일중으로 완료해주세요" → due: "내일중", due_date: (현재날짜+1)
   - 전문: "다음주 수요일까지 제출" → due: "다음주 수요일까지", due_date: (다음주 수요일)
   - 전문: "이번주 금요일까지" → due: "이번주 금요일까지", due_date: (이번주 금요일)
   - 전문: "12월 20일까지" → due: "12월 20일까지", due_date: "2024-12-20"
   - 마감일 없음 → due: null, due_date: null

7. 근거 없는 정보 생성 금지.
8. 위 json의 구성요소(description, assignee, due, due_date)는 반드시 포함한다.
9. 출력은 JSON 한 덩어리로. 부가 텍스트 금지.
10. json 형태에 들어가는 내용은 절대 추측 없이 전문 기반으로 작성.
