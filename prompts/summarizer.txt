[지시]
[입력 전문]만을 근거로 "agendas" 필드를
요청된 **JSON 형식**으로만 출력하라.
**다른 설명이나 텍스트, 마크다운(` ```json `)을 포함하지 말고, 순수한 JSON 객체 하나만 생성해야 한다.**

[입력 전문]
{transcript}

[출력 형식 (JSON)]
아래 JSON 구조를 정확히 따르되, 실제 값은 전문 기반으로 채워라.

{{
  "agendas": [
    {{
      "agenda_id": "agenda_1",
      "agenda": "안건 제목",
      "agenda_description": "안건 설명",
      "summary": {{
        "who": "누가 (참여자/담당자)",
        "what": "무엇을 (안건 내용/결정사항)",
        "when": "언제 (시간/기한)",
        "where": "어디서 (장소/환경)",
        "why": "왜 (목적/이유)",
        "how": "어떻게 (방법/절차)",
        "how_much": "얼마나 (비용/규모)",
        "how_many": "몇 개 (수량)",
        "agenda_summary": "위 5W3H 기반으로 작성한 줄글 요약"
      }}
    }},
    {{
      "agenda_id": "agenda_2",
      "agenda": "안건 제목",
      "agenda_description": "안건 설명",
      "summary": {{
        "who": "...",
        "what": "...",
        "when": "...",
        "where": "...",
        "why": "...",
        "how": "...",
        "how_much": "...",
        "how_many": "...",
        "agenda_summary": "..."
      }}
    }}
  ]
}}

[규칙]
1. 반드시 유효한 JSON 객체 하나만 출력한다.

2. agendas는 개수 상관없이 전문에서 추출할 수 있는 모든 안건을 뽑는다.

3. agenda_id:
   - "agenda_1", "agenda_2", "agenda_3" 형식으로 순차 부여
   - 반드시 포함 필수

4. agenda 및 agenda_description:
   - agenda: 안건의 제목/주제 (전문에서 추출)
   - agenda_description: 안건에 대한 상세 설명 
   - **전문에 나오는 단어만 사용**, 근거 없는 표현 생성 금지

5. summary (5W3H 구조화):
   각 항목은 **전문에 언급된 내용만** 추출. 언급 없으면 null 설정.

   - who: 이 안건과 관련된 사람/팀/담당자 (예: "test1, test2", "개발팀")
   - what: 안건의 핵심 내용/결정사항/목표 (예: "신규 기능 개발", "예산 승인")
   - when: 시간/일정/기한 (예: "이번 주 금요일까지", "12월 20일")
   - where: 장소/환경/플랫폼 (예: "회의실 A", "온라인", "서버 환경")
   - why: 목적/이유/배경 (예: "사용자 요청 대응", "성능 개선")
   - how: 방법/절차/프로세스 (예: "API 개발 후 테스트", "단계별 진행")
   - how_much: 비용/예산/규모 (예: "500만원", "3개월 소요")
   - how_many: 수량/개수 (예: "3개 기능", "5명 투입")
   - agenda_summary: 위 5W3H 항목을 기반으로 작성한 **자연스러운 줄글 요약**
     * 전문에 언급된 5W3H 요소들을 조합하여 2-4문장으로 작성
     * 없는 항목은 요약에 포함하지 않음

6. summary 작성 예시:
   [전문 예시]
   "test1: 이번 스프린트에서 쿠버네티스 클러스터 구축을 완료해야 합니다.
    test2: 네, 저와 test3이 담당하겠습니다. 이번 주 금요일까지 완료 목표입니다.
    test1: 좋습니다. 성능 개선이 목적이니 단계별로 진행해주세요."

   [출력 예시]
   {{
     "agenda_id": "agenda_1",
     "agenda": "쿠버네티스 클러스터 구축",
     "agenda_description": null,
     "summary": {{
       "who": "test2, test3",
       "what": "쿠버네티스 클러스터 구축 완료",
       "when": "이번 주 금요일까지",
       "where": null,
       "why": "성능 개선",
       "how": "단계별로 진행",
       "how_much": null,
       "how_many": null,
       "agenda_summary": "test2와 test3이 성능 개선을 위해 쿠버네티스 클러스터 구축을 담당하며, 이번 주 금요일까지 단계별로 진행하여 완료할 예정입니다."
     }}
   }}

7. 근거 없는 정보 생성 절대 금지.
8. 출력은 JSON 한 덩어리로. 부가 텍스트 금지.
9. json 형태에 들어가는 내용은 절대 추측 없이 전문 기반으로 작성. 

위 규칙을 모두 지켜 JSON 결과만 출력하라.
